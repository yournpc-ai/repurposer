## 2. 核心流程逐步还原

```
上传素材 → 自动转写（说话人分离 + 帧级时间码）→ 对话式指令编辑 → 随时手动接管 → 导出
```

1. **输入**：浏览器直接上传（MP4/MOV 等）；自动转写（100+ 语言、说话人分离、词级时间码）
2. **编辑（核心交互）**：自然语言指令——"去掉所有口头禅""把 4:20 到 5:10 剪掉""把这 3 小时播客做成 highlight reel"——每条指令执行为**真多轨时间线上的真实可编辑操作**（视频轨/图形轨/音乐轨），可继续下指令也可随时手动接管
3. **文本编辑**：删转写文字 = 删对应音视频
4. **审阅**：云端项目跨端同步（浏览器/桌面/agent 面）
5. **导出**：MP4（16:9/9:16，**上限 1080p**）；XML 交接 Premiere/DaVinci、SRT/VTT 字幕文件（官方页面确认）；ProRes 仅"limited 1080p"，4444 alpha 未证实 ⚠️；**全档位无水印**

## 4. 深拆：对话式编辑的交互模型

- **每条指令 = 时间线上可检查、可修改、可撤销的真实操作**，不是一次性模板渲染——这是它与"prompt-to-video"（Factory 范式）的本质区别，也是专业用户敢用的前提
- **AI 与手动无缝切换**：用户可随时接管时间线手改，AI 再在改动后的状态上继续工作——"copilot 而非 autopilot"
- **指令范围**：去静默/口头禅、selects 组装、节奏收紧、highlight reel、reframe、平台版本化——全是"剪辑师日常会说的话"
- **对我们的意义**：我们 project-scoped chat（PRD §5.1，指令派发后台 run）与其同向。差距与机会：
  - 它把指令绑定到**时间线操作**；我们的指令绑定到**生成/渲染 run**——我们的抽象更适合"零剪辑技能"用户（教授/专家），它的更适合半专业用户
  - 值得吸收：**指令结果的可撤销/可检查性**必须是第一等公民；每条指令执行前应给成本预估（联动透明计费）

## 5. 深拆：agent 接入面（MCP 原生）

- 无公开 REST API，而是**原生 MCP server（OAuth 登录）+ ChatGPT/Codex 桌面插件**（开源仓库 `ChatCut-Inc/agent-plugin`）：让外部 agent 直接驱动编辑器，编辑结果以真实可编辑时间线返回
- 2026-07-09 Codex 插件上线——"在 Codex 里拥有一个完整 AI 视频编辑器"
- **趋势信号**：它把"被 agent 调用"当作主分发面而非补充。我们的 4-layer agent 架构（GenerationContext → Content Director → Executors → Reviser）天然适合暴露 MCP 工具面，P2 方向

## 6. 信任与合规观察

- 小团队（2-10 人）+ 上海远程工程力量；用户数据（80,000+ 创作者、25,000 Discord 成员）为自述 ⚠️
- 隐私政策无 GDPR 专章——欧洲机构采购场景同样缺位
- 全档位无水印 + Free 档可商用导出，获客打法激进

## 7. 一句话总结

ChatCut 是"AI 编辑"的下一交互形态的样本：指令即操作、操作可撤销、agent 即分发面。它的聚焦（只做编辑、不做发布）反过来印证了我们"生成 + 分发"全链路的差异化空间。
