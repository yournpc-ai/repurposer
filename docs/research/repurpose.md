## 2. 核心流程

**Workflow = 源 → 转换 → 目的地** 的自动化规则：
- **源**：YouTube/播放列表/Shorts、TikTok、Instagram、Facebook、播客 RSS、Zoom、Webex、Google Drive、Dropbox、Twitch、SoundCloud
- **转换**：横转竖模板、audiogram（声波纹烧录）、自动修剪至平台时长、烧录源字幕、AI 写发布文案（GPT-4o）
- **目的地（15+）**：TikTok、YouTube、IG Reels/Stories、FB、**LinkedIn 个人+公司页**、X、Threads、Pinterest、Snapchat、Bluesky、Amazon Video、SoundCloud
- **自动化**：轮询源 → 自动发布（排期/回填）或人工审核队列；日历视图；平台保护限速（3 视频/小时）

## 3. 关键观察（对我们的意义）

1. **LinkedIn 个人 + 公司页**：七家中唯二支持（另一家 Opus 仅企业号）——验证了我们核心渠道的可行性与稀缺性
2. **零内容智能**：无 AI 切条、无打分、无生成——分发自动化可脱离内容智能独立成立，但也正是它的天花板
3. **平台关系是长期维护成本**：同平台 workflow 2026-06 被迫下线、被迫加限速——做分发就是长期与平台 API 政策共舞
4. **审核队列可选**：机构合规刚需的正确形态，我们应默认人工确认
5. **借鉴优先级**：workflow 规则模型（源→目的地）、审核队列、限速保护、不按量计费；不借鉴其"同内容原样铺全平台"（我们的 agent 生成时即按目的地分化内容）

## 4. 一句话总结

Repurpose 是我们做 LinkedIn 直发时唯一真正的参照系：它的 workflow 规则与审核队列值得学，它的"零智能搬运"正是我们"生成时感知目的地"的超车点。
